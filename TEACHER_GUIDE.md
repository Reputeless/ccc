# TEACHER GUIDE

## このガイドについて

`CCC` は、教員がオリジナルのコース（問題集）を作成し、自分のサーバーでホストして運用できるオンラインジャッジです。  
教員は、問題ごとのフォルダに `problem.json`、`body.md`、`guide.md`、`01.in.txt` / `01.out.txt` などの問題ファイルを配置して、コースを構成できます。

このガイドでは、`CCC` の導入、ローカルでの確認、サーバーへのアップロード、問題の追加、検査、カスタマイズなど、教員向けの運用方法を説明します。  
学生向けの使い方や内部実装の詳細よりも、実際の運用に必要な情報を中心にまとめています。

サーバーにアップロードして使うためのファイル一式は、[GitHub リポジトリ](https://github.com/Reputeless/ccc) の [Releases](https://github.com/Reputeless/ccc/releases) からダウンロードできます。

`CCC` は、授業中に一斉提出させる用途よりも、学生が自習や復習で使う演習環境としての利用に向いています。

## 最短で使い始める

まずは次の順で進めてください。

1. サーバーの PHP バージョンが `8.2` 以上か確認します
2. 必要なファイルとフォルダをサーバーへアップロードします
3. ブラウザで `validate.php` を開き、問題の状態を確認します
4. 必要に応じて `problems/` の下に新しい問題フォルダを追加します
5. 問題一覧ページと個別ページで表示を確認します

この順で進めると、Web 開発に詳しくなくても基本的な運用を始めやすくなります。

## 導入前の確認

### PHP バージョン

- `CCC` は `PHP 8.2` 以上を前提にしています
- 少なくとも `PHP 5.x` では動作しません
- さくらのレンタルサーバでは、コントロールパネルで対象ドメインの PHP バージョンを確認してください

### インターネット接続

- `CCC` の判定機能は、コードの実行に外部サービス `Wandbox` を利用します
- そのため、判定にはインターネット接続が必要です
- 学校や組織のネットワークで利用する場合は、事前に判定できるか確認しておくと安心です

## ローカルでプレビューする

- `CCC` は、公開サーバーにアップロードする前に、手元の PC で動作確認できます
- 問題文の見え方、解説、問題一覧への表示、`publishedAt` の挙動などを事前に確認したいときに便利です

### PHP の簡易サーバーを起動する

コマンドプロンプトやターミナルを開き、`CCC` のフォルダ（プロジェクトのルート）へ移動してから、次のコマンドを実行します。

```sh
php -S localhost:8000
```

その後、ブラウザで次の URL を開きます。

```text
http://localhost:8000/
```

### ブラウザで開く

- 問題一覧: `http://localhost:8000/`
- 教師用ガイド: `http://localhost:8000/teacher-guide.php`
- 問題ステータス: `http://localhost:8000/validate.php`

### よくある注意

- `index.html` をファイルとして直接開くのではなく、PHP の簡易サーバーを使ってください
- Python の簡易サーバーでは PHP が動かないため、設定や API の読み込みに失敗します
- `publishedAt` を未来日時にした問題は、ローカルでも問題一覧に表示されません
- CLI で作成した問題も、この方法ですぐ確認できます

## 新しい問題を追加する（手作業）

- `problems/` の下に新しいフォルダを作成します
- フォルダ名がそのまま問題 ID になります
- 問題 ID には、半角英数字とハイフン（例: `sample-001`）を使うと安全です
- 問題 ID は、小文字を使うことをおすすめします
- 日本語、空白、記号の多い名前は避けてください
- 問題フォルダには、まず次のようなファイルを用意します

```text
problems/
  └─ sample-001/         ← フォルダ名が問題 ID になります
       ├─ problem.json   ← 設定ファイル
       ├─ body.md        ← 問題文
       ├─ guide.md       ← 解説（任意）
       ├─ 01.in.txt      ← 項目 1 の入力側ファイル
       └─ 01.out.txt     ← 項目 1 の正解側ファイル
```

| ファイル | 役割 |
|---|---|
| `problem.json` | 問題種別、問題番号、タイトル、講義回、難易度、`publishedAt` などの設定を書きます |
| `body.md` | 問題文を Markdown で書きます。見出し・箇条書き・番号付きリスト・引用・水平線・太字・斜体・打ち消し線・コードブロック・表・画像が使えます |
| `guide.md` | 解説を Markdown で書きます（任意） |
| `01.in.txt` | 最初の項目を書きます。`code` 問題では入力例、`text` 問題では設問文や入力欄ラベルになります |
| `01.out.txt` | 最初の正解側データを書きます。`code` 問題では正しい出力、`text` 問題では許容解答一覧になります |

- `problem.json`、`body.md`、`guide.md` は、`UTF-8（BOM なし）` で保存してください

### `problem.json` の主な項目

| 項目 | 説明 |
|---|---|
| `type` | 問題種別です。コーディング問題では `code`、文章入力問題では `text` を指定します |
| `number` | 問題番号です。現在のテンプレートでは初期値として `"N/A"` を入れています |
| `title` | 問題タイトルです。現在のテンプレートでは初期値として `"無題"` を入れています |
| `lecture` | 講義回です。整数で指定します。未設定にしたい場合は `null` を使います |
| `difficulty` | 難易度です。`1` から `3` の整数で指定します。未設定にしたい場合は `null` を使います |
| `profileId` | 問題の文脈を表すプロファイルです。コーディング問題では使用する言語に合った値を指定します。文章入力問題でも、必要に応じて `c23` や `python3.14` などを指定できます |
| `publishedAt` | 公開日時です。たとえば `2026-04-01T09:00:00+09:00` のように書きます。未指定なら常に公開されます |

- コーディング問題では `type` に `code` を使います
- 文章入力問題では `type` に `text` を使います
- `profileId` には、その問題で使う言語や文脈に合った値を指定します
- 文章入力問題でも、必要に応じて `profileId` を使えます
- `guide.md` がない場合でも問題は動作します
- 問題ページのタイトルは Markdown の外で表示されるため、`body.md` は通常 `## 問題` から書き始めると自然です
- ひな形が必要な場合は `templates/default/` を参照してください

### `01.in.txt` / `01.out.txt` などの意味

ファイル名は、`01` から `06` までの固定連番です。途中を飛ばさずに置いてください。

コーディング問題（`type: "code"`）:

- `01.in.txt`
  - 最初の入力例を書きます
- `01.out.txt`
  - 最初の正しい出力を書きます

文章入力問題（`type: "text"`）:

- `01.in.txt`
  - 最初の小問の設問文や入力欄ラベルを書きます
- `01.out.txt`
  - 最初の小問で許容する解答を、1 行 1 個で書きます

たとえば、`,` の読み方を答えさせる場合は、`01.out.txt` を次のように書けます。

```text
カンマ
コンマ
```

### `publishedAt` について

- `problem.json` に `publishedAt` を書くと、その日時までは問題一覧に表示されません
- 指定しない場合は常に公開扱いです
- 形式は `2026-04-01T09:00:00+09:00` のようなタイムゾーン付き日時を推奨します
- これは一覧や API 側の公開制御であり、厳密なアクセス制限ではありません
- `問題ステータス` からはタイトルなどのメタデータを確認できます
- また、`problems/` 配下のファイルは URL が分かれば直接参照できる可能性があります

## 新しい問題を追加する（CLI）

CLI は、コマンドを入力して使う方法です。  
ローカルで CLI を使える場合は、次のコマンドで新しい問題フォルダを作成できます。

```sh
php tools/create.php sample-001
```

このコマンドは `problems/sample-001/` を作成し、`templates/default/` から次のファイルをコピーします。

- `problem.json`
- `body.md`
- `guide.md`
- `01.in.txt`
- `01.out.txt`

現在の `create.php` は、コーディング問題（`type: "code"`）向けの雛形を作成します。文章入力問題（`type: "text"`）を作る場合は、生成後に `problem.json` と `01.in.txt` / `01.out.txt` などを手で調整してください。

必要に応じて、次のオプションも指定できます。

- `--template <name>`
  - `templates/` の下にあるテンプレート名を指定します。通常は `default` を使います
- `--profile <id>`
  - `problem.json` の `profileId` を上書きします。たとえば `c23`, `cpp23`, `python3.14` などを指定できます

## 問題を検査する（問題ステータスページ）

- フッタの `問題ステータス` から開けます
- 問題一覧に載らない問題や、足りないファイルを確認しやすいです
- まずはこちらで確認するのが手軽です

## 問題を検査する（CLI）

CLI は、コマンドを入力して使う方法です。  
ローカルで CLI を使える場合は、次のコマンドで問題データを検査できます。

```sh
php tools/validate.php
php tools/validate.php sample-001
```

- 引数なし:
  - 全問題を検査します
- `<problem-id>` 付き:
  - その問題だけを検査します
- `Error` が 1 件でもあると終了コード `1` を返します
- `Warning` だけなら終了コードは `0` です

## 学習記録の扱い

- 学習記録と入力内容は、サーバーではなく学生それぞれのブラウザに保存されます
- 別の端末や別のブラウザへは自動では引き継がれません
- ブラウザデータの削除、端末変更、ブラウザ再インストールなどで失われる可能性があります
- 必要に応じて、学生に学習記録のエクスポートでバックアップを取るよう案内すると安心です
- 入力内容は学習記録とは別に扱われ、同じブラウザ内にだけ残ります
- `入力内容` と `解いた記録・理解度` は別々に消去できます

## FAQ（公開・アクセス）

### Q. `教師用ガイド` は学生にも見えますか？

見えます。`教師用ガイド` は `CCC` 本体に含まれる公式ページであり、認証付きの非公開ページではありません。

### Q. `問題ステータス` から何が見えますか？

問題 ID、問題番号、タイトル、講義回、難易度、`profileId`、`publishedAt`、`items`、`guide` の有無、検査結果などが見えます。

### Q. `publishedAt` を使えば、特定日時まで完全に非公開にできますか？

完全にはできません。`publishedAt` は一覧や API 側での公開制御であり、厳密なアクセス制限ではありません。詳しくは上の `publishedAt` の説明を参照してください。

### Q. URL を知っていれば未公開問題にアクセスできますか？

可能性があります。`CCC` の未公開制御は表示制御であり、認証つきの厳密な非公開機能ではありません。

## FAQ（問題作成・検査）

### Q. CLI を使わなくても問題は追加できますか？

できます。`problems/` の下に問題フォルダを作り、必要なファイルを手作業で追加してください。

### Q. `guide.md` は必須ですか？

必須ではありません。なくても問題は動作します。

### Q. `lecture` や `difficulty` は必須ですか？

必須ではありません。未設定にしたい場合は `null` を使えます。

### Q. `difficulty` にはどんな値を書けますか？

`1` から `3` の整数を書きます。表示ラベルは通常 `基本 / 標準 / 発展` です。

### Q. `profileId` には何を書けばよいですか？

その問題で使う言語や文脈に合った値を書きます。たとえば、C の問題なら `c23`、C++ の問題なら `cpp23`、Python の問題なら `python3.14` のように使えます。文章入力問題でも、問題の文脈を示したいときは `profileId` を指定できます。文脈を特に示したくない場合は省略できます。

### Q. 対応言語は何ですか？

現在の標準対応は次のとおりです。

| `profileId` | 表示名 |
|---|---|
| `c17` | `C17` |
| `c23` | `C23` |
| `cpp20` | `C++20` |
| `cpp23` | `C++23` |
| `python3.14` | `Python 3.14` |

### Q. 講義回の表示を `第 N 章` に変えられますか？

変えられます。`config/app.json` の `lectureLabelTemplate` を編集してください。

### Q. 問題文で画像を使えますか？

使えます。問題フォルダの中に画像ファイルを置き、`body.md` から Markdown の画像記法で参照してください。

### Q. `01.in.txt` / `01.out.txt` などは何組まで置けますか？

`01` から `06` まで、最大 6 組です。

### Q. `01` を飛ばして `02` から始めてもよいですか？

できません。`01.in.txt` / `01.out.txt` などのファイルは、`01` から順に連番で置く必要があります。

### Q. 問題が一覧に出てきません。

`number` や `title` の不足、`problem.json` の不正、必要な項目ファイル不足などがあると一覧から外れます。まず `問題ステータス` または `php tools/validate.php` で確認してください。

### Q. 公開した問題を修正できますか？

できます。同じ問題フォルダのまま修正すれば、学生の `解けた` 状態や理解度の記録はそのまま残ります。typo 修正や説明の改善はそのまま更新して問題ありません。

### Q. 公開した問題を削除できますか？

削除できます。問題一覧や個別ページからは見えなくなりますが、学生側のブラウザに残っているローカル記録は自動では消えません。

### Q. 問題フォルダ名を変えるとどうなりますか？

フォルダ名がそのまま問題 ID なので、別問題として扱われます。既存の URL やローカル保存キーにも影響します。

## FAQ（文章入力問題）

### Q. 文章入力問題では何を判定しますか？

各小問について、学生が入力した解答が `01.out.txt` / `02.out.txt` などに書かれた許容解答のどれかに一致するかを判定します。

### Q. 文章入力問題の解答はどこで判定されますか？

ブラウザ内で判定します。コーディング問題のように `Wandbox` へ送信して実行することはありません。

### Q. 文章入力問題では複数の正解を登録できますか？

できます。たとえば `01.out.txt` に複数行を書くと、そのどれでも正解になります。

```text
カンマ
コンマ
```

### Q. 文章入力問題の正解は学生に見えてしまいますか？

見えてしまいます。文章入力問題の判定はブラウザ内で行うため、許容解答はブラウザに配信されます。秘匿したい知識問題には向いていません。

### Q. 文章入力問題で大文字小文字や全角半角の違いは無視されますか？

現在は無視されません。前後の空白だけを除去して比較します。

### Q. `create.php` で文章入力問題は作れますか？

現在の `create.php` はコーディング問題向けの雛形を作成します。文章入力問題を作る場合は、生成後に `problem.json` を `type: "text"` に変更し、`01.in.txt` / `01.out.txt` などを手で調整してください。

## FAQ（学習記録）

### Q. 学習記録はどこに保存されますか？

学習記録と入力内容は、サーバーではなく学生それぞれのブラウザに保存されます。

### Q. 教員が学生の学習記録を集めることはできますか？

現在の `CCC` には、学習記録をサーバーへ集める機能はありません。必要に応じて、学生にエクスポートした JSON ファイルを提出してもらう運用は可能です。

### Q. 別の PC や別のブラウザでも同じ記録を使えますか？

自動では引き継がれません。必要に応じて、学習記録のエクスポート / インポートを案内してください。

### Q. 学生がブラウザデータを消すとどうなりますか？

そのブラウザに保存されていた学習記録や入力内容は失われます。

### Q. シークレットモードや共有 PC でも学習記録は残りますか？

残らない場合があります。シークレットモード（プライベートブラウズ）では、ブラウザを閉じたときに学習記録が消えることがあります。共有 PC でも、再起動時やログアウト時にブラウザデータが初期化される設定だと記録は残りません。

### Q. 入力内容も引き継がれますか？

入力内容は学習記録とは別に扱われます。現在はエクスポート / インポート対象ではなく、同じブラウザ内にだけ残ります。

## FAQ（コードの判定）

### Q. 判定できないことがあります。

Wandbox（コードの実行に使っている外部サービス）側の混雑や制限の影響で、一時的に判定できないことがあります。時間を置いて再試行してください。

授業中に多人数が同時に判定すると、エラーになりやすくなることがあります。放課後や自習時間での利用の方が安定しやすいです。

### Q. 隠れテストケースは使えますか？

使えません。現在の `CCC` は、問題文に表示している公開の入出力例だけで判定します。授業用の自習ツールとして、学生が手元で原因を調べやすいことを優先しています。

### Q. 出力が少し違っても正解になりますか？

途中の空白差は不正解です。ただし、末尾の空白と末尾改行の差は無視します。

### Q. 乱数や現在時刻を扱う問題は作れますか？

現在の完全一致判定では向いていません。乱数や時刻のように出力が毎回変わる問題には対応していません。
