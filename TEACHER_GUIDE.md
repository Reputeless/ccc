# TEACHER GUIDE

## このガイドについて

- このガイドは、教員が `CCC` を授業や自習用に運用するための手引きです
- 主に、導入前の確認、アップロード、問題追加、問題検査、学習記録の扱いをまとめています
- 学生向けの使い方ではなく、教員が知っておくと便利な運用情報に絞っています
- 内部構成や仕様の概要は、必要に応じて [README.md](./README.md) を参照してください

## 導入前の確認

### PHP バージョン

- `CCC` は `PHP 8.2` 以上を前提にしています
- 少なくとも `PHP 5.x` では動作しません
- さくらのレンタルサーバでは、コントロールパネルで対象ドメインの PHP バージョンを確認してください

## ローカルでプレビューする

- `CCC` は、公開サーバにアップロードする前に、手元の PC で動作確認できます
- 問題文の見え方、解説、問題一覧への表示、`publishedAt` の挙動などを事前に確認したいときに便利です

### PHP の簡易サーバを起動する

プロジェクトのルートで、次のコマンドを実行します。

```powershell
php -S localhost:8000
```

その後、ブラウザで次の URL を開きます。

```text
http://localhost:8000/
```

### ブラウザで開く

- 問題一覧: `http://localhost:8000/`
- 教師用ガイド: `http://localhost:8000/teacher-guide.php`
- 問題ステータス: `http://localhost:8000/validate.php`

### よくある注意

- `index.html` をファイルとして直接開くのではなく、PHP の簡易サーバを使ってください
- Python の簡易サーバでは PHP が動かないため、設定や API の読み込みに失敗します
- `publishedAt` を未来日時にした問題は、ローカルでも問題一覧に表示されません
- CLI で作成した問題も、この方法ですぐ確認できます

## 表示文言をカスタマイズする

- 一方で、`問題ステータス` や `教師用ガイド` のような公式ツール部分は、原則として固定です

## 新しい問題を追加する（手作業）

- `problems/` の下に新しいフォルダを作成します
- フォルダ名がそのまま問題 ID になります
- 次のファイルを用意します

| ファイル | 役割 |
|---|---|
| `problem.json` | 問題番号、タイトル、講義回、難易度、`profileId` などの設定を書きます |
| `body.md` | 問題文を Markdown で書きます。見出し・箇条書き・コードブロック・表・画像が使えます |
| `guide.md` | 解説を Markdown で書きます（任意） |
| `01.in.txt` | 最初の入出力例の入力を書きます |
| `01.out.txt` | 最初の入出力例の正しい出力を書きます |

### `problem.json` の主な項目

| 項目 | 説明 |
|---|---|
| `number` | 問題番号です。現在のテンプレートでは初期値として `"N/A"` を入れています |
| `title` | 問題タイトルです。現在のテンプレートでは初期値として `"無題"` を入れています |
| `lecture` | 講義回です。整数で指定します。未設定にしたい場合は `null` を使います |
| `difficulty` | 難易度です。`1` から `3` の整数で指定します。未設定にしたい場合は `null` を使います |
| `profileId` | 使用する言語プロファイルです。通常は `c23`、必要に応じて `cpp23` や `python3.14` などを使います |
| `publishedAt` | 公開日時です。未指定なら常に公開されます |

- 入出力例は `01.in.txt` / `01.out.txt` から始め、必要なら `02`、`03` と連番で追加します
- 入出力例は最大 `06.in.txt` / `06.out.txt` まで使えます
- `guide.md` がない場合でも問題は動作します
- 問題ページのタイトルは Markdown の外で表示されるため、`body.md` は通常 `## 問題` から書き始めると自然です
- 制約欄では、`<=` ではなく `≤` を使うと見やすくなります
- ひな形が必要な場合は `templates/default/` を参照してください

### `publishedAt` について

- `problem.json` に `publishedAt` を書くと、その日時までは問題一覧に表示されません
- 指定しない場合は常に公開扱いです
- 形式は `2026-04-01T09:00:00+09:00` のようなタイムゾーン付き日時を推奨します
- これは一覧や API 側の公開制御であり、厳密なアクセス制限ではありません
- `問題ステータス` からはタイトルなどのメタデータを確認できます
- また、`problems/` 配下のファイルは URL が分かれば直接参照できる可能性があります

## 新しい問題を追加する（CLI）

ローカルで CLI を使える場合は、次のコマンドで新しい問題フォルダを作成できます。

```powershell
php tools/create.php sample-001
```

このコマンドは `problems/sample-001/` を作成し、`templates/default/` から次のファイルをコピーします。

- `problem.json`
- `body.md`
- `guide.md`
- `01.in.txt`
- `01.out.txt`

必要に応じて、次のオプションも指定できます。

- `--template <name>`
- `--profile <id>`

## 問題を検査する（問題ステータスページ）

- フッタの `問題ステータス` から開けます
- 問題一覧に載らない問題や、足りないファイルを確認しやすいです
- まずはこちらで確認するのが手軽です

## 問題を検査する（CLI）

ローカルで CLI を使える場合は、次のコマンドで問題データを検査できます。

```powershell
php tools/validate.php
php tools/validate.php sample-001
```

- 引数なし:
  - 全問題を検査します
- `<problem-id>` 付き:
  - その問題だけを検査します
- `Error` が 1 件でもあると終了コード `1` を返します
- `Warning` だけなら終了コードは `0` です

## 学習記録の扱い

- 学習記録とコード入力内容は、サーバではなく学生それぞれのブラウザに保存されます
- 別の端末や別のブラウザへは自動では引き継がれません
- ブラウザデータの削除、端末変更、ブラウザ再インストールなどで失われる可能性があります
- 必要に応じて、学生に学習記録のエクスポートでバックアップを取るよう案内すると安心です
- コード入力内容は学習記録とは別に扱われ、同じブラウザ内にだけ残ります
- `コード入力内容` と `解いた記録・理解度` は別々に消去できます

## FAQ（公開・アクセス）

### Q. `教師用ガイド` は学生にも見えますか？

見えます。`教師用ガイド` は `CCC` 本体に含まれる公式ページであり、認証付きの非公開ページではありません。

### Q. `問題ステータス` から何が見えますか？

問題 ID、問題番号、タイトル、講義回、難易度、`profileId`、`publishedAt`、入出力例の状態、`guide` の有無、検査結果などが見えます。

### Q. `publishedAt` を使えば、特定日時まで完全に非公開にできますか？

完全にはできません。`publishedAt` は問題一覧や API 側での公開制御には使えますが、厳密なアクセス制限ではありません。`問題ステータス` からはタイトルなどのメタデータを確認できますし、`problems/` 配下のファイルも URL が分かれば直接参照できる可能性があります。

### Q. URL を知っていれば未公開問題にアクセスできますか？

可能性があります。`CCC` の未公開制御は一覧や API での表示制御であり、認証つきの厳密な非公開機能ではありません。

## FAQ（問題作成・検査）

### Q. CLI を使わなくても問題は追加できますか？

できます。`problems/` の下に問題フォルダを作り、必要なファイルを手作業で追加してください。

### Q. `guide.md` は必須ですか？

必須ではありません。なくても問題は動作します。

### Q. `lecture` や `difficulty` は必須ですか？

必須ではありません。未設定にしたい場合は `null` を使えます。

### Q. `difficulty` にはどんな値を書けますか？

`1` から `3` の整数を書きます。表示ラベルは通常 `基礎 / 中級 / 発展` です。

### Q. `profileId` には何を書けばよいですか？

通常は `c23` を使います。必要に応じて `cpp23` や `python3.14` を使えます。

### Q. 対応言語は何ですか？

現在の標準対応は次のとおりです。

| `profileId` | 表示名 |
|---|---|
| `c17` | `C17` |
| `c23` | `C23` |
| `cpp20` | `C++20` |
| `cpp23` | `C++23` |
| `python3.14` | `Python 3.14` |

### Q. 講義回の表示を `第 N 章` に変えられますか？

変えられます。`config/app.json` の `lectureLabelTemplate` を編集してください。

### Q. 問題文で画像を使えますか？

使えます。問題フォルダの中に画像ファイルを置き、`body.md` から Markdown の画像記法で参照してください。

### Q. 入出力例は何個まで置けますか？

`01` から `06` まで、最大 6 組です。

### Q. `01` を飛ばして `02` から始めてもよいですか？

できません。入出力例は `01` から順に連番で置く必要があります。

### Q. 問題が一覧に出てきません。

`number` や `title` の不足、`problem.json` の不正、入出力例ファイル不足などがあると一覧から外れます。まず `問題ステータス` または `php tools/validate.php` で確認してください。

### Q. 公開した問題を修正できますか？

できます。同じ問題フォルダのまま修正すれば、学生の `解けた` 状態や理解度の記録はそのまま残ります。typo 修正や説明の改善はそのまま更新して問題ありません。

### Q. 公開した問題を削除できますか？

削除できます。問題一覧や個別ページからは見えなくなりますが、学生側のブラウザに残っているローカル記録は自動では消えません。

### Q. 問題フォルダ名を変えるとどうなりますか？

フォルダ名がそのまま問題 ID なので、別問題として扱われます。既存の URL やローカル保存キーにも影響します。

## FAQ（学習記録）

### Q. 学習記録はどこに保存されますか？

学習記録とコード入力内容は、サーバではなく学生それぞれのブラウザに保存されます。

### Q. 教員が学生の学習記録を集めることはできますか？

現在の `CCC` には、学習記録をサーバへ集める機能はありません。必要に応じて、学生にエクスポートした JSON ファイルを提出してもらう運用は可能です。

### Q. 別の PC や別のブラウザでも同じ記録を使えますか？

自動では引き継がれません。必要に応じて、学習記録のエクスポート / インポートを案内してください。

### Q. 学生がブラウザデータを消すとどうなりますか？

そのブラウザに保存されていた学習記録やコード入力内容は失われます。

### Q. コード入力内容も引き継がれますか？

コード入力内容は学習記録とは別に扱われます。現在はエクスポート / インポート対象ではなく、同じブラウザ内にだけ残ります。

## FAQ（判定）

### Q. 判定できないことがあります。

Wandbox 側の混雑や制限の影響で、一時的に判定できないことがあります。時間を置いて再試行してください。

### Q. 隠れテストケースは使えますか？

使えません。現在の `CCC` は、問題文に表示している公開の入出力例だけで判定します。授業用の自習ツールとして、学生が手元で原因を調べやすいことを優先しています。

### Q. 出力が少し違っても正解になりますか？

途中の空白差は不正解です。ただし、末尾の空白と末尾改行の差は無視します。

### Q. 末尾改行や末尾空白は無視されますか？

はい。末尾の空白と末尾改行の差は無視します。

### Q. 乱数や現在時刻を扱う問題は作れますか？

現在の完全一致判定では向いていません。乱数や時刻のように出力が毎回変わる問題には対応していません。
